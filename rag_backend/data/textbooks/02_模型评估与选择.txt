# 第二章 模型评估与选择

## 2.1 经验误差与过拟合

训练误差（Training Error）：模型在训练集上的误差，也称经验误差。
泛化误差（Generalization Error）：模型在未知数据上的期望误差。

过拟合（Overfitting）：模型在训练集上表现很好，但在测试集上表现差。原因是模型过于复杂，学到了训练数据中的噪声和细节，而没有学到普遍规律。

欠拟合（Underfitting）：模型在训练集上表现也不好，说明模型过于简单，未能学到数据的真实规律。

泛化能力（Generalization Ability）：模型在未见过的数据上的预测能力，是衡量学习算法好坏的核心指标。

## 2.2 评估方法

留出法（Hold-out）：将数据集划分为两个互斥的子集——训练集和测试集。常见比例是 70%/30% 或 80%/20%。为了减少随机性带来的偏差，通常进行多次随机划分取平均。

交叉验证（Cross Validation）：将数据集分成 k 个大小相似的互斥子集（k 折），每次用 k-1 个子集的并集作为训练集，余下的那个子集作为测试集。这样可获得 k 组训练/测试集，进行 k 次训练和测试，最终取 k 次结果的均值。最常用的是 10 折交叉验证。

自助法（Bootstrapping）：给定包含 m 个样本的数据集 D，对它进行有放回的随机采样，得到一个包含 m 个样本的训练集 D'。D 中约有 36.8% 的样本未出现在 D' 中，这些样本可作为测试集。自助法特别适用于数据集较小的情况。

## 2.3 性能度量

### 分类任务度量

混淆矩阵（Confusion Matrix）：展示分类结果中各类别正确和错误预测数量的矩阵，包含四个基本量：
- TP（True Positive）：真正例，实际为正且预测为正
- TN（True Negative）：真负例，实际为负且预测为负
- FP（False Positive）：假正例，实际为负但预测为正
- FN（False Negative）：假负例，实际为正但预测为负

准确率（Accuracy）：分类正确的样本占总样本的比例。公式：Accuracy = (TP + TN) / (TP + TN + FP + FN)

精确率（Precision）：预测为正的样本中真正为正的比例。公式：Precision = TP / (TP + FP)。精确率关注的是"预测为正的有多少是对的"。

召回率（Recall）：真正为正的样本中被正确预测为正的比例。公式：Recall = TP / (TP + FN)。召回率关注的是"实际为正的有多少被找到了"。

F1 分数（F1 Score）：精确率和召回率的调和平均数。公式：F1 = 2 × P × R / (P + R)。F1 综合了精确率和召回率，当两者都很高时 F1 才高。

ROC 曲线（ROC Curve）：以假正率（FPR = FP/(FP+TN)）为横轴、真正率（TPR = TP/(TP+FN)）为纵轴绘制的曲线。曲线越靠近左上角，模型性能越好。

AUC（Area Under ROC Curve）：ROC 曲线下的面积。AUC = 0.5 表示随机猜测，AUC = 1.0 表示完美分类。通常 AUC > 0.8 被认为性能良好。
